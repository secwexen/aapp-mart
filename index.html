Why AAPP-MART?

AAPP-MART stands out from traditional offensive security tools in its approach:

  • Traditional scanners → static, reactive, often limited to known vulnerabilities.
  • BAS (Breach & Attack Simulation) tools → rely on predefined playbooks and limited scenarios.
  • AAPP-MART → predictive, autonomous, and adaptive: forecasts attack paths and executes intelligent multi-agent simulations.

By combining AI-driven attack path prediction with autonomous red team simulations, AAPP-MART provides organizations with a forward-looking security posture, not just reactive alerts.

Example Usage

Run the engine against a target after installation:

from aapp_mart.core.orchestrator import AAPP_MART

engine = AAPP_MART(target="192.168.1.10")
engine.run()
report = engine.get_report()
print(report)

Command-line interface (CLI) example:

# Run a simulation
$ aapp-mart --target 192.168.1.10 --mode full

# Generate report
$ aapp-mart --report latest

This will execute the engine and generate actionable attack path predictions and security recommendations. (GitHub)

Release Notes & Roadmap

Development Phases

The development of AAPP-MART is organized in three strategic phases:

  • Phase 1 – Initial public release: v0.1.0 Pre-release is the first public release of AAPP-MART, establishing the project foundation. Includes initial project skeleton, Quick Start example, installation instructions, testing setup, and ethical use guidelines. Core functionality is still under development.
  • Phase 2 – Added new modules, agents, files and examples: v0.2.0 Pre-release adds new modules, agents, example scripts, and updated ML training files. Core simulation engine and attack orchestration are still in development. Includes minor fixes and improvements. Intended for testing and early contributions.
  • Phase 3 – Core Stabilization: v0.3.0 Pre-release focuses on stabilizing AAPP-MART’s core components. Core Simulation Engine and MART Agent Framework are stabilized, AI attack path prediction improved, ML files updated, with minor bug fixes and CLI/API improvements. For testing and early contributions.
